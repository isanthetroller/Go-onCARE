1. Technology Stack & Architecture

Desktop Application

Python 3 PyQt6 (GUI Framework) MySQL Database Fusion Style Engine
  • Frontend: PyQt6 — cross-platform native desktop UI with custom QSS theming
  • Backend: Python Mixin architecture — modular backend classes (AuthMixin, PatientMixin, EmployeeMixin, ClinicalMixin, etc.) composed into a single AuthBackend class
  • Database: MySQL with parameterized queries for SQL injection prevention
  • Design: Teal (#388087) / Soft Sage (#F6F6F2) / Accent (#BADFE7) color palette with consistent card-based layout, gradient headers, and modern form styling
  • Error Handling: Global exception hook catches any unhandled errors and displays them to the user instead of silently crashing

2. User Roles & Access Control

Go-onCARE uses role-based access control (RBAC). Each user account is assigned a role that determines which pages, features, and data they can access.

Role Primary Responsibilities Key Accessible Pages
Admin Full system access — manages everything: patients, appointments, employees, services, settings, user accounts Dashboard, Patients, Appointments, Clinical, Employees, HR, Payroll, Analytics, Settings, Activity Log
Receptionist Front desk operations — registers patients, schedules appointments, creates invoices Dashboard, Patients, Appointments, Clinical (Queue + Billing), Activity Log
Doctor Clinical care — views their own patients, completes queue entries, records consultations Dashboard (doctor-specific), My Patients, Clinical Queue, My Analytics
Nurse Triage — calls waiting patients from the queue, records vital signs (BP, weight, height, temp) Dashboard, Clinical Queue (triage-focused)
HR Human resources — manages employee records, approves/declines leave requests, processes paychecks Dashboard, Employee Management, Leave Requests, Payroll
Finance Financial oversight — approves/rejects paycheck requests before disbursement Dashboard, Payroll Approval Queue

3. Authentication & Security

Login System

  • Email + password authentication against the users table
  • Role-based redirection — each role sees a tailored interface upon login
  • Login activity is automatically logged in the audit trail

First-Login Password Change

  • Newly created accounts are flagged with must_change_password = 1
  • The system forces the user to change their password on first login before granting access
  • This flag is cleared once the password is successfully changed

Account Management (Admin Only)

  • Create accounts — Admin can create user accounts with name, email, password, and role assignment
  • Reset passwords — Admin can reset any user's password; the user must change it on next login
  • Delete accounts — Admin can remove user accounts from the system
  • Change own password — Any user can change their own password (requires current password verification)

Auto Clock-In / Clock-Out

  • When an employee logs in, the system automatically records a clock-in in the attendance table
  • When they log out (or close the application), it records a clock-out
  • This provides automatic time tracking without manual attendance entry

5. Patient Management

Add New Patient

  • Required fields: Full Name, Phone, Address
  • Validated fields: Phone (digits only, with +63 prefix), Email (format check), Name (min 2 characters)
  • Personal info: Sex, Date of Birth (modern calendar picker), Civil Status, Emergency Contact
  • Medical info: Blood Type (A+/−, B+/−, AB+/−, O+/−), Notes
  • Conditions: Multi-select checkboxes from a configurable list of standard conditions
  • Discount: Selectable discount type (e.g., PWD, Senior Citizen) with an associated discount percentage
  • ID Proof: If the selected discount requires ID proof, the system requires an uploaded image before saving
  • Duplicate Detection: Before saving, the system checks for existing patients with the same name, phone, or email
  • Review & Confirm: A styled summary dialog shows all entered data for verification before final save

Edit Patient

  • Same form as Add, pre-populated with existing data
  • Updates all fields including conditions, discount, ID proof, and status

Patient Profile Viewer

  • Detailed profile dialog with tabbed sections:
  • Information Tab: All personal, medical, and contact details in a card layout
  • Appointments Tab: Full history of appointments with dates, doctors, services, and statuses
  • Invoices Tab: All invoices with amounts, payment status, and payment methods
  • Queue History Tab: Past queue entries with times, purposes, and statuses

Patient Table & Search

  • Searchable patient list with columns: Name, Phone, Email, Blood Type, Discount, Conditions, Status, Last Visit
  • Real-time search filtering by name, phone, or email
  • Delete and Merge functionality available

6. Appointment Scheduling

Create Appointment

  • Select patient (searchable dropdown), doctor, service, date, and time
  • Modern calendar popup for date selection (same widget system-wide)
  • Walk-in support: Appointments scheduled for today are treated as walk-ins
  • Date validation: Cannot schedule appointments in the past or more than 90 days ahead
  • Optional notes, cancellation and reschedule reasons

Appointment Confirmation

  • Pending appointments can be confirmed by receptionist or admin
  • ID Proof validation: If the patient has a discount requiring ID proof (e.g., PWD) and no ID image has been uploaded, confirmation is blocked with a clear warning
  • Confirmed appointments are synced to the clinical queue on their appointment date

Appointment Status Flow

  • Pending → Confirmed → In Progress → Completed
  • Can also be Cancelled with a reason, or Rescheduled
  • Appointments are automatically cancelled if the assigned doctor goes on approved leave

7. Clinical Operations & Queue

Clinical Queue Management

  • Auto-sync: Today's confirmed appointments are automatically added to the queue each day
  • Call Next: Doctors and nurses can call the next patient from the queue
  • Priority system: Doctors prioritize "Triaged" patients (already seen by nurse) before "Waiting" patients
  • Nurse triage: Nurses only pick "Waiting" patients; they record vitals and mark the patient as "Triaged"
  • Status flow: Waiting → Triaged (by nurse) → In Progress (by doctor) → Completed

Vital Signs (Triage)

  • Blood Pressure (required) — Systolic / Diastolic with real-time validation (anti-debug: systolic must be > diastolic)
  • Weight (optional) — in kilograms
  • Height (optional) — supports both cm and feet with automatic conversion between units
  • Temperature (optional) — in degrees
  • Vitals are saved to the queue entry and displayed to the doctor when they see the patient

Doctor's Patient View

  • Doctors see their own assigned patients with triage results (vitals tab with dates)
  • Can view full patient profile, history of appointments, and previous vitals
  • Queue entries the nurse triaged appear in the doctor's queue for follow-up

8. Billing & Invoicing

Create Invoice

  • Patient selection: Searchable dropdown of active patients
  • Appointment link: Optional — link the invoice to a specific appointment
  • Multi-item support: Add multiple services as line items, each with its own price
  • Automatic discount: If the patient has a discount type (e.g., PWD 20%), it is automatically applied to all line items

  • Notes: Optional invoice notes

Auto-Invoice from Queue

Invoice Management

  • Status tracking: Unpaid → Partial → Paid (or Voided)
  • Payment recording: Record partial or full payments against an invoice
  • Void invoices: Admin can void incorrect invoices
  • Invoice detail view: Shows patient info, line items, payment method, and amounts

10. HR & Leave Management

Request Leave (All Employees)

  • Any employee can submit a leave request from their dashboard
  • Specify Leave From date, Leave Until date, and Reason
  • Modern calendar date picker (same widget as DOB and appointment date pickers)
  • Validation: Leave From must be at least tomorrow; Leave Until must be after Leave From
  • Requests start with "Pending" status

HR Leave Administration

  • HR sees all pending leave requests in a management queue
  • Approve: Sets employee status to "On Leave," records leave dates, and auto-cancels all the doctor's confirmed appointments during the leave period
  • Decline: HR provides a decline reason/note
  • Notifications: The employee receives an in-app notification about the decision

My Leave Requests

  • Employees can view their own leave history with status (Pending, Approved, Declined), HR notes, and timestamps

11. Payroll & Paycheck Processing

Paycheck Request Lifecycle

  1. HR creates paycheck request — selects employee, period (from/to), gross amount; system auto-calculates deductions
  2. Finance reviews — Approve (with optional note) or Reject (with reason)
  3. HR disburses — marks approved paycheck as disbursed
  4. Employee notified — in-app notification at each stage

Deduction Calculation

  • System automatically calculates standard deductions based on gross salary
  • Net amount = Gross - Deductions
  • Full paycheck history maintained per employee

Paycheck Statuses

  • Pending → awaiting finance review
  • Approved → approved by finance, awaiting HR disbursement
  • Rejected → declined by finance with reason
  • Disbursed → payment released to employee

14. Data Validation & Safeguards

Input Validation

  • Patient Name: Minimum 2 characters
  • Phone: Digits only, validated format with +63 prefix
  • Email: Regex format validation
  • Address: Required field for new patients
  • Blood Pressure: Systolic must be greater than diastolic (anti-debug validation)
  • Appointment Date: Must be today or within 90 days; cannot be in the past
  • Leave Dates: Leave From ≥ tomorrow; Leave Until ≥ Leave From + 1 day
  • Service Prices: Custom price validator for numeric input

Business Logic Safeguards

  • Discount ID Proof: If a discount type requires ID proof and no image is uploaded, the system blocks both new patient saves and appointment confirmations
  • Duplicate Patient Detection: Alerts the user when adding a patient with a matching name, phone, or email
  • Leave Auto-Cancellation: When a doctor's leave is approved, all their confirmed/pending appointments during the leave period are automatically cancelled
  • Invoice Discount Enforcement: Discount percentages are resolved from the database at invoice creation time — not trusted from the UI — preventing manipulation
  • Paycheck Approval Workflow: Paychecks require finance approval before HR can disburse, ensuring dual authorization
